# User Stories

1) **Intake & sources**
As a user, I can drag/drop folders, single photos, and URLs into a project, so I can build a photo basis without reorganizing originals.

2) **Non-destructive processing**
As a user, the app keeps originals in place and runs background jobs (thumbnailing, metadata extraction, clustering) with clear progress and resume, so I can keep working while it processes.

3) **Cross-source clustering**
As a user, I get automatic event/theme clusters that span multiple devices and folders, so photos of the same event are grouped together regardless of source.

4) **Duplicate stacks + best shots**
As a user, I can view burst/near-duplicate stacks and mark one or multiple "best shots," so I can keep the right frames without reviewing every image.

5) **Staging area + cross-theme access**
As a user, I have a staging area that can show photos from other themes, so I can pull images across themes when assembling pages.

6) **Book structure & pages**
As a user, I can create chapters/themes, reorder them, set page counts, and drag photos onto pages in a responsive layout, so I can shape the book quickly.

7) **Reuse & text elements**
As a user, I can reuse a photo in multiple chapters and add text elements to pages, so the story and captions are flexible.

8) **Preview + export/reference**
As a user, I can preview the book and export a structured list of photos/texts by page (or keep it as a reference), so I can reproduce it in a print service.
